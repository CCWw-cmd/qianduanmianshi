# 初始前端工程化

## 由来

早期前端开发较为简单

现在:组件化、模块化、es6、css 预处理器的兼容、代码质量与规范、图片压缩、CI/CD 等

因此，我们需要一套规范、标准、自动的工程实践，这就是工程化

> 前端 CI: 产品代码持续集成到远程仓库。集成前后自动化执行一些任务。如集成前进行代码 lint 规范、commit message 规范、代码质量、单元测试。通过集成后自动打包构建项目。

> 前端 CD: 持续部署/交付。即将构建后的代码自动化部署到远程服务器、发布到 NPM 等。

## 目的

降本增效提质

## 涉及方面

模块化：将代码拆分为独立的模块，便于复用和维护。
组件化：通过封装组件提高开发效率。
自动化：减少手动操作，提高开发和部署效率。
规范化：统一代码风格和开发流程。

## 相关技术

- Nodejs（node版本管理nvm、包管理工具pnpm、包源管理nrm）

- 构建工具与模块化（webpack、vite、rollup、esbuild）

- Vue、React 相关技术栈

- Lint 规范化工具（eslint、stylelint）

- CI/CD（linux、docker、k8s、verdaccio）

- 服务端技术（部署相关：云服务器、域名、nginx、https、CDN；开发相关：redis、mysql、express/midwayjs、ORM-sequelize、pm2、日志等）

- 单元测试

1. 基础技术
   Node.js：前端工程化的基石，用于构建工具和服务端开发。
   包管理工具：pnpm、npm、yarn。

2. 构建与打包工具
   构建工具：webpack、vite、rollup、esbuild。
   任务运行器：gulp。

3. 前端框架与技术栈
   框架：Vue、React。
   状态管理：Redux、Vuex、Pinia、MobX。
   路由管理：React Router、Vue Router。

4. 代码规范与质量控制
   代码检查工具：eslint、stylelint。
   代码格式化工具：prettier。
   提交规范工具：commitlint、lint-staged、husky。

5. 持续集成与部署（CI/CD）
   持续集成（CI）：代码质量检查、单元测试、自动打包。
   持续部署（CD）：自动化部署到远程服务器或发布到 NPM。
   工具与平台：GitHub Actions、Jenkins、GitLab CI/CD。
   容器化技术：Docker、Kubernetes（k8s）。
   操作系统：Linux。

6. 服务端与运维技术
   缓存与数据库：Redis、MySQL。
   Web 服务器：Nginx。
   域名与云服务：域名解析、阿里云、腾讯云、AWS。

## 前端工程化的由来

早期的前端开发较为简单，主要是静态页面开发，功能需求较少，代码结构也较为简单。然而，随着前端技术的发展，出现了以下复杂需求：

组件化：前端开发逐渐转向基于组件的开发模式。
模块化：需要将代码拆分为独立的模块，便于复用和维护。
现代化技术：如 ES6、CSS 预处理器等，需要解决兼容性问题。
代码质量与规范：需要统一代码风格，保证代码质量。
性能优化：如图片压缩、资源优化等。
自动化流程：如 CI/CD，自动化构建、测试和部署。
这些需求促使前端开发从简单的页面开发转向更复杂、更规范的工程化实践。

## 前端工程化的概念

前端工程化是一套规范、标准和自动化的工程实践，旨在提升开发效率、保证代码质量，并实现高效的项目交付。它包括以下核心理念：

模块化：将代码拆分为独立的模块，便于复用和维护。
组件化：通过封装组件提高开发效率。
自动化：减少手动操作，提高开发和部署效率。
规范化：统一代码风格和开发流程。

## 前端工程化的目标

前端工程化的核心目标是：

降本：减少开发成本。
增效：提升开发效率。
提质：保证代码质量。
